## 项目概述
- **项目名称**: JYRiskManage (江阴风险管理平台)
- **项目类型**: Spring MVC + iBatis + Kafka
- **JDK版本**: JDK 1.7
- **服务器**: Tomcat 6
- **数据库**: Oracle + MySQL

## 一、环境准备

### 1. JDK 1.7 安装与配置
1. 下载 JDK 1.7 (项目中使用的是 jdk1.7.0_79)
2. 安装到指定目录，例如：`C:\Program Files\Java\jdk1.7.0_79`
3. 配置环境变量：
   - `JAVA_HOME`: `C:\Program Files\Java\jdk1.7.0_79`
   - `PATH`: 添加 `%JAVA_HOME%\bin`
   - `CLASSPATH`: `.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar`
4. 验证安装：打开CMD，执行 `java -version`

### 2. Tomcat 6 安装与配置
1. 下载 Apache Tomcat 6.x (建议 6.0.53 最终版本)
2. 解压到指定目录，例如：`D:\apache-tomcat-6.0.53`
3. 配置环境变量（可选）：
   - `CATALINA_HOME`: `D:\apache-tomcat-6.0.53`

## 二、IntelliJ IDEA 配置步骤

### 1. 导入项目
1. 打开 IntelliJ IDEA
2. 选择 `File` -> `Open`
3. 选择项目目录：`D:\Code\gongan\JYRiskManage`
4. 等待 IDEA 自动识别项目结构

### 2. 配置 JDK
1. `File` -> `Project Structure` -> `Project`
2. `Project SDK`: 选择 JDK 1.7
3. `Project language level`: 选择 `7 - Diamonds, ARM, multi-catch etc.`
4. 点击 `Apply`

### 3. 配置 Modules
1. `File` -> `Project Structure` -> `Modules`
2. 确认 `src` 目录被标记为 `Sources`
3. 确认 `WebRoot` 为 Web 资源目录

### 4. 配置 Tomcat
1. 点击右上角 `Add Configuration...`
2. 点击 `+` -> `Tomcat Server` -> `Local`
3. 配置如下：
   - **Name**: Tomcat 6 - JYRiskManage
   - **Application server**: 点击 `Configure`，选择 Tomcat 6 安装目录
   - **VM options** (重要): 
     ```
     -Dfile.encoding=UTF-8
     -Xms256m
     -Xmx512m
     -XX:PermSize=128m
     -XX:MaxPermSize=256m
     ```
   - **JRE**: 选择 JDK 1.7

4. 切换到 `Deployment` 标签：
   - 点击 `+` -> `Artifact` -> 选择 `JYRiskManage:war exploded`
   - 或点击 `+` -> `External Source` -> 选择 `WebRoot` 目录
   - **Application context**: 设置为 `/` 或 `/JYRiskManage`

5. 点击 `Apply` 和 `OK`


## 四、配置文件说明

### 1. 数据库配置
文件位置：`WebRoot/WEB-INF/jdbc.properties`

根据实际情况修改数据库连接信息。

### 2. Spring配置文件
- `applicationContext.xml`: Spring主配置

- `business-servlet.xml`: Spring MVC配置

文件位置：`WebRoot/WEB-INF/log4j.properties`

- `http://localhost:8080/` 或

主要入口页面：

- 首页: `index.jsp`

## 六、常见问题解决

### 1. 端口冲突
修改 Tomcat 配置文件 `conf/server.xml`，修改端口号（默认8080）

### 2. 编码问题
确保所有配置文件和IDE编码设置为 UTF-8

### 3. 数据库连接失败
- 检查数据库服务是否启动
- 检查 `jdbc.properties` 配置是否正确
- 检查网络连接和防火墙设置
- 确保 Oracle 驱动 `ojdbc14.jar` 存在

### 4. 内存溢出
增大 JVM 内存设置：
```
-Xms512m -Xmx1024m -XX:PermSize=256m -XX:MaxPermSize=512m
```

### 5. JAR包冲突
检查 `WebRoot/WEB-INF/lib` 目录，删除重复或冲突的JAR包

### 6. Kafka连接问题
如果不使用Kafka功能，保持 `web.xml` 中 Kafka 配置注释状态

## 八、开发建议

1. **版本控制**: 建议使用Git进行版本管理
2. **配置分离**: 将敏感配置（数据库密码等）与代码分离
3. **日志管理**: 定期清理和归档日志文件
4. **代码规范**: 遵循Java编码规范
5. **安全加固**: 修改默认密码，增强系统安全性
